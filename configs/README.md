# Configuration Files

This directory contains automatically generated configuration files for the LLM Fallbacks library. These files are updated daily at 12:00 AM UTC via [GitHub Actions](../.github/workflows/daily-config-update.yml).

## Files

### `free_models.json`

Array of free model objects sorted by `quality_score` (descending). Each entry includes:

```json
{
  "id": "gemini/gemini-2.0-flash",
  "provider": "gemini",
  "mode": "chat",
  "is_free": true,
  "input_cost_per_token": 0,
  "output_cost_per_token": 0,
  "context_length": 1000000,
  "max_output_tokens": 8192,
  "supports_vision": true,
  "supports_function_calling": true,
  "supports_tool_choice": true,
  "supports_response_schema": true,
  "supports_system_messages": true,
  "supports_audio_input": false,
  "supports_audio_output": false,
  "supports_pdf_input": false,
  "supports_prompt_caching": true,
  "quality_score": 85.26,
  "quality_source": "heuristic_v1"
}
```

Local-only models (ollama, vllm, lmstudio, xinference) are excluded.

### `free_models_ids.txt`

Plain text file with one model ID per line, in the same quality-score order as `free_models.json`. Useful for simple downstream consumption:

```bash
curl -s https://raw.githubusercontent.com/bolabaden/llm_fallbacks/main/configs/free_models_ids.txt | head -10
```

### `all_models.json`

Complete map of `model_id → spec` for all models known to LiteLLM (including paid models). Useful for debugging and analysis.

### `custom_providers.json`

Serialised configurations for custom providers (OpenRouter, VertexAI, Yandex).

### `litellm_config.yaml` / `litellm_config_free.yaml`

Ready-to-use LiteLLM proxy configurations:

```bash
# Run LiteLLM proxy with free models only
litellm --config configs/litellm_config_free.yaml
```

## Generation

These files are generated by:

```bash
python -m llm_fallbacks.generate_configs --output-dir configs
```

## Update Schedule

- **Automatic**: Daily at 12:00 AM UTC via GitHub Actions
- **Manual**: Trigger via GitHub Actions workflow dispatch, or run locally
- All changes are automatically committed and tracked in version control

## Quality Scoring

The `quality_score` in `free_models.json` is a transparent capability heuristic (0-100) based on context window size, function calling support, vision, tool choice, and other observable features. See the [main README](../README.md#quality-scoring) for the full scoring breakdown.

## Web UI

The `free_models.json` file is consumed by the [Chat UI](../docs/index.html) (deployed to GitHub Pages) to populate its model dropdown. The UI fetches the latest model list from the repo's raw URL on every page load.
